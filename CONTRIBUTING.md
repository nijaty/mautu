# Руководство для контрибьюторов MAUTU

Спасибо, что хотите помочь проекту! 🎉

## Что можно добавлять

- Новые мануалы по любым IT-темам
- Исправления ошибок и неточностей
- Улучшения существующих мануалов
- Переводы из других источников (с указанием источника)

## Структура файлов

Каждый мануал — это отдельный `.md` файл в соответствующей папке.

Пример имени файла: `docker-compose-basics.md`

## Шаблон мануала

```markdown
# Название

> Краткое описание в одну строку

## Что это и зачем

## Установка

## Основные команды / Примеры

## Типичные проблемы и решения

## Полезные ссылки
```

## Процесс проверки

1. Открываете Pull Request
2. Один из мейнтейнеров проверяет контент
3. При необходимости просят правки
4. После одобрения — merge!

## Вопросы?

Открывайте Issue с меткой `question`.
