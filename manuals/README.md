# Руководства MAUTU

В этой папке хранятся все IT-руководства для приложения MAUTU.
Каждый может внести вклад, добавив или отредактировав `.md`-файлы и открыв Pull Request.

## Структура папок

Организуйте руководства по категориям и подкатегориям:

```
manuals/
  <Категория>/
    <Подкатегория>/
      ваше-руководство.md
```

### Существующие категории

```
manuals/
  DevOps/
    Git/
    Ansible/
    Terraform/
    Kubernetes/
    Docker/
  Network/
    TCP-IP/
  Development/
    Backend/
      Python/
    Frontend/
      React/
  SysAdmin/
    Linux/
      Ubuntu/
      RHEL/
    Windows/
  DataBase/
  CyberSecurity/
  QA/
```

## Написание руководства

Каждый `.md`-файл должен начинаться с заголовка первого уровня — он становится названием руководства:

```markdown
# Название руководства

## Обзор

Краткое описание темы.

## Раздел

Содержимое раздела. Можно использовать:
- **Жирный текст**
- `встроенный код`
- Блоки кода (с указанием языка)

```bash
echo "пример команды"
```

## Ссылки

- Ссылка на официальную документацию
```

## Использование изображений

Поместите изображения в подпапку `images/` рядом с `.md`-файлом:

```
manuals/
  DevOps/
    Docker/
      images/
        architecture.png
      docker-intro.md
```

В файле руководства укажите относительный путь:

```markdown
![Описание](images/architecture.png)
```

Пути к изображениям автоматически преобразуются в абсолютные при импорте.

## Как внести вклад

1. Сделайте форк репозитория
2. Добавьте или отредактируйте `.md`-файл в нужной папке категории
3. Откройте Pull Request
4. После слияния PR пайплайн CI/CD автоматически собирает новый `.deb`-релиз

## Правила

- Используйте существующие папки категорий. Чтобы добавить **новую** категорию, сначала обсудите это в Issue.
- Одно руководство — один файл.
- Имена файлов должны быть в нижнем регистре через дефис: `docker-networking.md`
- Язык написания — русский.
